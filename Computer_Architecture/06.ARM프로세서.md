# ARM 프로세서
```
프로세서 = CPU(중앙처리장치)
```

<br>

# ARM
```
- Advanced RISC Machine
- RISC가 핵심(Reduced Instruction Set Computing)
- 복잡한 것보다 단순한 게 더 효율적임
- 임베디드 기기에 주로 사용되는 32비트 프로세서
```

<br>

# 구조
```
- ARM은 칩 기본 설계 구조만 만들고, 실제 기능 추가 및 최적화는 개별 반도체 제조사가 담당.
- 물리적 설계는 같지만 명령 집합이 다름 -> 서로 다른 칩이 될 수 있음
- 단순한 명령 집합 -> 적은 수의 트랜지스터가 필요함 -> 간결한 설계, 더 작은 크기
- 크기가 작고 전원 소모가 낮은 ARM CPU가 스마트폰이나 태블릿 PC같은 모바일 기기에 많이 사용됨.
```

<br>

# 장점
```
- x86은 성능이 최적화되도록 설계됨. ARM 기반 프로세서는 더 작은 크기로 비용을 줄이고 전력소비를 낮추고
  열발생도 줄이고 속도를 높이고 베터리 수명을 늘림
- ARM 아키텍처를 사용하면 하드웨어 설계자의 제어 권한이 커짐
  (ARM사가 직접 반도체를 제조하는게 아니라 반도체 회사에서 설계한 프로세서를 바탕으로 여러 기능을 추가할 수 있어서)
- ARM을 위해 개발된 프로세서는 오직 ARM 프로세서가 탑재된 기기에만 실행 가능함
- x86 CPU기반 프로그램은 ARM기반 기기에서 실행 불가능(별도 수정 필요)
- 하나의 ARM기기에서 동작하는 OS는 다른 ARM기반 기기에서도 잘 동작함
```
