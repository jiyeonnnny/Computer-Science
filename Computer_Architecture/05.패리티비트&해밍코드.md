# 패리티비트
## 패리티비트 개념
```
정보 전달 과정에서 오류 발생 여부를 확인하는 비트임.
전송하고자 하는 데이터의 각 문자에 1비트를 추가로 전송함. 
오류 검출 및 데이터 재전송 요청만 가능함.
```

<br>

## 패리티비트 종류(홀수/짝수)
```
홀수 패리티비트
- 전송하고자 하는 데이터 비트 + 패리티 비트의 1개수가 홀수가 되도록하는 패리티비트
- 전송하고자 하는 데이터 비트 1의 개수가 홀수이면 0, 짝수이면 1.

짝수 패리티비트
- 전송하고자 하는 데이터 비트 + 패리티 비트의 1개수가 짝수가 되도록하는 패리티비트
- 전송하고자 하는 데이터 비트 1의 개수가 짝수이면 0, 홀수이면 1.
```
```
Q. 홀수 짝수 왜 정하지? 왜 굳이 1비트를 패리티 비트로 사용하는거지?
데이터 송수신 과정에서, 알 수 없는 요인으로 비트값이(하나만) 틀어졌을때 이 오류를 검출할 수 있음.

- 수신된 데이터 전체 비트와 패리티 비트를 계산함으로써 데이터 오류 발생 여부를 알 수 있음.
- 단점 : 오류발생여부만 알 수 있고, 어디서 오류난 건지 모름
```

<br>

## 패리티비트 예시
```
1. 100 전송하려는 데이터. 100을 2진수로 변환 1100100.
2. 1100100이란 데이터를 보낼때 뒤에 1비트 추가해서 8비트 전송함.
3. 짝수 패리티일 경우, 11001001값을 전송
```

# 해밍코드
## 해밍코드 개념
```
해밍코드를 사용한다면 수신자는 오류검출 뿐만 아니라 오류 수정도 가능함
```

<br>

## 해밍코드 특징
```
1. 2비트 오류를 검출할 수 있고, 1비트 오류를 교정할 수 있음
2. 데이터 비트 외에 오류 검출 및 교정을 위한 잉여 비트가 많이 필요함
3. 해밍코드 중 1,2,4,8,16,...2^N번째 비트는 오류 검출을 위한 패리티비트임.
4. 패리티 규칙(짝수 혹은 홀수)가 정해졌을때, N번째 패리티비트는 N번째 비트에서 시작해서 N비트만큼 포함하고,
N비트씩 건너뛴 비트들을 대상으로, 패리티 비트를 결정할 수 있음
```

<br>

## 해밍코드 방법
```
2^N 번째 자리인 1,2,4번째(2의 0,1,2승) 자릿수가 패리티 비트임.
이 숫자로부터 시작해서 세개의 패리티비트가 짝수인지 홀수인지 기준으로 판별함.

Q. 짝수 패리티의 해밍코드가 0011011일때, 오류가 수정된 코드는?
N번째 패리티비트는 N번째에서 시작함. N비트 만큼 포함. N비트씩 건너띈 비트들을 대상으로
패리티 비트 범위를 지정하고 각 패리티 비트를 결정함.

1번 경우 1 3 5 7
2번 경우 2 3 6 7
4번 경우 4 5 6 7

2^N 자리가 패리티비트라고 가정하면(짝수 패리티)
1. 1 3 5 7번째 비트 확인 : 0101로 짝수이므로 0
2. 2 3 6 7번째 비트 확인 : 0111로 홀수이므로 1
3. 4 5 6 7번째 비트 확인 : 1011로 홀수이므로 1

역순으로 패리티비트 110을 도출. 10진법으로 바꾸면 6. 6번째 비트 수정하면 됨.
따라서 정답은 0011001
```
